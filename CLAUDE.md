# Preferences

## Responses
- Be concise — say what changed and what's next, nothing more
- No bullet-point summaries of every small thing done
- Just say "Done" when complete unless there's something worth flagging

## Making changes
- Always test visually before reporting done (screenshot for UI, smoke test for APIs)
- Show a screenshot so I can see the result without opening a browser
- Mobile-first: fit everything on a single iPhone screen, no scrolling required

## Git & deployment
- After every change: commit with a clear message, push to the active branch, deploy
- Always provide the live URL after deploying
- Keep commit messages short and descriptive

## Project files
- Always maintain a README.md with a play/use link, how it works, and tech notes

## Code documentation
- Always include docstrings/doc comments on functions and classes to help team members debug
- Add inline comments for non-obvious logic
